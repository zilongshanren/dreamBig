爆款游戏监控与 IAA 改造分析平台 PRD v1.0
1. 产品概述
1.1 产品名称
GamePulse AI
副标题：爆款游戏监控 + IAA 改造顾问平台

1.2 产品定位
一个面向游戏发行、买量、立项、变现团队的 7×24 小时自动监控与分析平台，持续追踪 Steam、Google Play、App Store、TapTap 等平台上的游戏榜单、口碑、社交反馈、版本变化，并自动输出：

爆款预警
竞品追踪
玩法机制拆解
IAA 化适配建议
立项参考结论
1.3 一句话价值
帮助团队更早发现爆款、更快理解玩法、更稳判断是否适合做 IAA / Hybrid Monetization，减少“靠拍脑袋立项”和“错过趋势”的损失。

2. 产品目标
2.1 业务目标
平台核心要解决 4 个问题：

发现爆款太晚

榜单起来了才知道，错过窗口期
知道火，但不知道为什么火

不知道核心机制、爽点、传播点在哪里
知道想抄，但不知道怎么改成 IAA

广告位放哪、什么机制适合激励视频、什么不该碰
信息太散，人工看不过来

榜单、评论、社媒、社区、版本更新都分散在不同平台
2.2 产品目标
MVP 阶段达成以下目标：

每日自动扫描重点平台游戏榜单与高热游戏
自动识别“值得关注”的异常增长游戏
自动汇总评论、评分、社交反馈
自动输出游戏机制与 IAA 适配分析
通过飞书 / 企业微信 / 邮件完成分级推送
2.3 成功指标
MVP 上线后 1~2 个月内关注：

爆款预警准确率 ≥ 60%
人工分析耗时下降 ≥ 70%
每周产出有效候选游戏数 ≥ 20
业务团队实际点击查看战报率 ≥ 40%
预警后进入人工深度评估池的比例 ≥ 20%

11. 数据与算法策略
11.1 分析主逻辑
平台不是单看绝对值，而是更关注：

增长速度
变化方向
多信号共振
持续性
11.2 核心评分
爆款预警分
text
HotScore =
  0.25 * RankingVelocity     // 14 天榜单线性回归动量
  + 0.20 * GenreFit           // 品类 IAA 适配度（查表）
  + 0.15 * SocialBuzz         // 7 天社媒声量加权
  + 0.15 * CrossPlatform      // 跨平台覆盖度
  + 0.10 * RatingQuality      // 评分 + 评论量综合
  + 0.10 * CompetitionGap     // 同品类已 IAA 化数量反比
  + 0.05 * AdActivity         // 14 天广告创意数量

说明（与 PRD v1.0 初版的偏离）：
- v1.0 初版的 5 维度（RankMomentum/ReviewVelocity/SocialVelocity/RatingHealth/UpdateImpact）在 Phase 1 实现时被重构为更精细的 7 维度。
- 新增 GenreFit：复用 shared/genres.json 的 iaa_score 查表，避免对所有品类一视同仁。
- 新增 CrossPlatform：游戏在多平台同时上线 = 通用吸引力强信号。
- 新增 CompetitionGap：同品类已 IAA 化数量越少，立项窗口越大。
- 新增 AdActivity：第三方已经在投同类游戏 = 市场已被验证。
- ReviewVelocity 被吸收进 RatingQuality（评论量是其分量之一）。
- UpdateImpact 暂未实现，规划中（依赖更新事件时间轴功能）。
- 实现版本字段：potential_scores.algorithm_version='v1'，未来变更需 bump。

IAA适配分
text
IAAFitScore =
0.30 * BreakpointDensity
+ 0.25 * ReplayStrength
+ 0.20 * ReviveNeed
+ 0.15 * MetaDepth
- 0.10 * MonetizationConflict

口碑健康分
text
ReputationScore =
0.40 * RatingLevel
+ 0.25 * PositiveTopicRatio
- 0.20 * NegativeTopicConcentration
+ 0.15 * ReviewTrend

19. 风险与限制
19.1 数据源限制
不同平台开放程度不同，竞品数据并不总能通过官方 API 完整拿到，所以平台需采用：

官方 API 优先
公开页面补充
第三方信号辅助
人工纠偏机制
19.2 平台规则风险
需要严格遵守各平台对数据访问、展示、使用方式的规则，避免：

非法抓取
频率过高
违规再分发
在不允许的场景中展示榜单
19.3 LLM 幻觉风险
AI 输出的“玩法拆解”和“IAA 建议”必须绑定证据源，不允许只给无依据结论。

19.4 跨平台同名映射风险
同名、重名、别名、不同地区名称不同，需保留人工修正机制。

20. 验收标准
MVP 验收以“闭环”而不是“功能数量”为准：

必须满足
能稳定抓取四个平台核心游戏信息
能自动识别榜单/评论异动
能给出单游戏战报
能输出 IAA 评估结论
能通过飞书/邮件推送
能支持人工搜索游戏并查看详情
验收示例
输入某款游戏名称，能在 1 分钟内返回其详情与分析
某款游戏 24 小时内快速爬榜时，系统能触发预警
战报能包含口碑、热度、玩法、IAA 四部分内容
